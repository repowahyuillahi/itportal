## PROMPT OPUS 4.6 — PAGE BASED (PAGES/TICKETS) FULL

Kamu adalah senior fullstack developer. Buat aplikasi **IT Helpdesk / Ticketing** dengan **PHP Native (tanpa framework)** dan UI **Tabler.io terbaru (Bootstrap 5)**. Fokus: **mudah dimaintenance oleh pemilik**, jadi gunakan **page-based routing** dengan folder `pages/`.

### Environment

* Local dev: **XAMPP (Apache + PHP)** di Windows
* Production: **aaPanel (Linux) PHP 8.3**
* Database: MySQL/MariaDB, gunakan DB dari file SQL yang saya lampirkan: `itportal (1).sql` (import ke MySQL)

### Aturan utama (wajib)

* **Tidak pakai MVC**.
* Setiap halaman berupa file PHP di folder `pages/...`.
* Logika umum dipindah ke `includes/` (helper) agar tidak duplikasi.
* Semua query DB wajib **PDO prepared statements**.
* Form POST wajib punya **CSRF token**.
* Output HTML wajib di-escape untuk anti XSS.
* Upload file aman (whitelist ext + size limit + rename).
* Tidak boleh bikin file kosong / placeholder. Semua file harus ada isinya dan runnable.

# 2) ROUTING (PAGE BASED) + URL REWRITE

Gunakan URL rapi seperti:

* `/login`
* `/logout`
* `/dashboard`
* `/tickets`
* `/tickets/create`
* `/tickets/view?id=123` (boleh) atau `/tickets/123` (opsional)

Minimal requirement: URL rewrite lempar semua ke `index.php`.

Buat `.htaccess`:

* jika file/folder ada -> langsung
* selain itu -> `index.php`

`index.php`:

* parse `$_SERVER['REQUEST_URI']`
* mapping ke file di `pages/`
* kalau tidak ada -> `pages/errors/404.php`

# 5) FITUR UTAMA

## A) AUTH

* Login memakai `users` table (gunakan kolom password hash yang ada; jika belum ada, tambahkan kolom `password_hash` atau gunakan kolom password yang sesuai dari SQL).
* Session-based auth.
* Middleware sederhana:

  * halaman selain `/login` wajib login kecuali `/track`

Role:

* user: buat ticket + lihat ticket milik sendiri
* staff: lihat & kelola ticket yang di-assign / semua ticket (pilih salah satu, jelaskan)
* admin: full access + user management

## B) TICKETS (USER)

Halaman:

* `pages/tickets/index.php`

  * list ticket milik user (admin/staff bisa lihat semua)
  * ada filter status + search ticket_code/subject
  * pagination
* `pages/tickets/create.php`

  * form: subject, category, priority, message awal, optional attachment
  * POST ke `pages/tickets/actions/create.php`
* `pages/tickets/view.php`

  * tampil detail + status badge + assigned staff
  * tampil timeline messages
  * form reply (POST ke `actions/reply.php`)
  * upload file optional

Ticket code:

* format: `TCK-YYYYMMDD-XXXX` (XXXX incremental per hari atau random 4 digit yang unik)

## C) TICKETS (STAFF/ADMIN)

Aksi:

* assign ticket ke staff (`actions/assign.php`)
* update status (`actions/update_status.php`)
* close ticket -> status `closed` atau `resolved`

RULE WAJIB:

* Jika status berubah ke `resolved` atau `closed`:

  * revoke token type `public_track` (set `revoked_at=NOW()`)
  * public link `/track?token=...` jadi tidak aktif (menampilkan pesan link kadaluarsa/tidak aktif)

## D) PUBLIC TRACKING LINK (TANPA LOGIN)

`pages/track/index.php`

* input: `?token=xxxxx`
* cari token di `ticket_tokens` type=public_track dan revoked_at IS NULL
* jika tidak valid -> tampilkan halaman “Link tidak aktif”
* jika valid -> tampilkan:

  * ticket_code, subject (opsional), status, timeline message (public)
* setelah ticket closed/resolved link otomatis invalid

Saat ticket dibuat:

* generate token random panjang (misal 32-64 chars)
* simpan sebagai `public_track` token
* tampilkan link di halaman ticket view & juga di list

## E) LINK UPDATE VIA WHATSAPP (STUB DULU)

Saat ticket dibuat juga generate token type `wa_update`.

Buat tombol “Kirim link update via WhatsApp” (untuk sekarang hanya generate link):

* WA link format:

  * `https://wa.me/<nomor>?text=<urlencode(pesan)>`
* Pesan minimal:

  * “Update ticket {ticket_code}: {APP_URL}/wa?token=xxxxx”
    Buat halaman stub:
* `pages/tickets/view.php` menampilkan “WA Update Link”
* Buat endpoint stub `pages/wa/index.php` atau pakai `pages/track/wa.php` (pilih salah satu rapi)
* Endpoint WA:

  * kalau token valid dan belum revoked:

    * tampilkan form “Kirim update (komentar)”
    * submit -> simpan ke `ticket_messages` sebagai pesan baru
  * ini untuk simulasi sebelum Baileys, jangan implement Baileys sekarang
* Sediakan `includes/NotificationService.php` sebagai stub (kosong) untuk integrasi Baileys nanti.
